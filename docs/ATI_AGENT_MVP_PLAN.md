# План реализации MVP ATI-агента ТехноЛогистики

## 1. Цель MVP

Сделать первый безопасный контур:

```text
тестовая заявка или MAX → парсинг → черновик ATI → запись результата → ручная проверка
```

На первом этапе агент ничего не публикует и никому не пишет автоматически.

## 2. Ограничения

- `DRY_RUN=true` включён по умолчанию.
- Никакой автопубликации на ATI.su.
- Никаких автоматических сообщений перевозчикам.
- Никаких изменений боевого Аюба.
- Никаких изменений сайта.
- Никаких реальных токенов в репозитории.

## 3. Этап 1 — каркас проекта

Готово в текущем репозитории:

- README.md;
- `.env.example`;
- requirements.txt;
- app/config.py;
- app/orchestrator.py;
- парсер заявок;
- генератор черновика ATI;
- заглушки интеграций ATI, MAX, Gmail, Google Sheets;
- пример заявки;
- тест парсера.

## 4. Этап 2 — локальный dry-run

Критерий готовности:

1. Агент читает `examples/sample_max_request.txt`.
2. Находит слово `ЗАЯВКА`.
3. Извлекает маршрут, авто, дату, состояние, оплату, комментарий.
4. Формирует `ati_draft`.
5. Печатает JSON в консоль.
6. Записывает событие в локальный журнал.
7. Не делает реальных внешних действий.

Команда:

```bash
python -m app.main
```

## 5. Этап 3 — Google Sheets

После настройки сервисного аккаунта:

1. Создать таблицу MVP.
2. Добавить вкладки: Заявки, Черновики ATI, Перевозчики, Ставки, Переговоры, Рыночная аналитика.
3. Реализовать запись заявки и черновика.
4. Не делать публикацию.

## 6. Этап 4 — MAX webhook

1. Создать отдельного MAX-бота для ATI-агента.
2. Добавить в тестовую группу.
3. Настроить HTTPS webhook.
4. Получать сообщения.
5. Обрабатывать только сообщения со словом `ЗАЯВКА`.
6. Записывать результат в Google Sheets.

## 7. Этап 5 — ATI.su API

До кода нужны реальные доступы ATI.su:

- API публикации грузов;
- API поиска транспорта;
- API мессенджера;
- API средних ставок;
- API прогноза цен.

Пока доступ не подтверждён, ATI Client остаётся заглушкой.

## 8. Этап 6 — ручное подтверждение

Перед любым боевым действием агент должен создать approval request:

- публикация ATI;
- отправка сообщения перевозчику;
- принятие ставки;
- изменение заявки;
- архивирование заявки.

## 9. Критерии успеха MVP

- Заявка корректно парсится.
- Черновик ATI формируется.
- Данные можно проверить человеком.
- Никакое боевое действие не выполняется без подтверждения.
- Аюб и сайт не затронуты.
