# Отчёт о реализуемости интеграции с ATI.su API

## 1. Предварительный вывод

ATI.su имеет официальный портал разработчиков и публичную документацию по нескольким API, которые потенциально подходят для ATI-агента:

- API для работы с грузами;
- API поиска транспорта;
- API мессенджера;
- API средних ставок;
- API ставок в похожих грузах / прогноза цен.

Но наличие документации не означает, что доступ уже есть у аккаунта ТехноЛогистики. Перед боевой разработкой нужно запросить подключение и проверить реальные права, лимиты и стоимость.

## 2. Что нужно запросить у ATI.su

Нужно написать в поддержку ATI.su и запросить доступ к API для:

1. публикации грузов;
2. редактирования грузов;
3. архивирования грузов;
4. поиска транспорта;
5. ATI Messenger;
6. средних ставок;
7. прогноза цен / ставок в похожих грузах.

Также нужно уточнить:

- какие API доступны текущему аккаунту;
- какие API платные;
- какие лимиты запросов;
- какие условия использования;
- можно ли использовать API для полуавтоматической работы с ручным подтверждением;
- разрешена ли автоматизированная отправка сообщений перевозчикам после подтверждения человека.

## 3. Что можно делать через API, если доступ подтвердится

### Публикация грузов

Потенциально: создавать черновики/публикации грузов на ATI.su на основе заявки из MAX.

Для MVP: только готовить черновик и показывать человеку.

### Поиск транспорта

Потенциально: искать подходящий транспорт и перевозчиков по маршруту, типу транспорта и дате.

Для MVP: сохранять найденных перевозчиков в Google Sheets.

### Мессенджер ATI

Потенциально: отправлять сообщения перевозчикам и читать ответы.

Для MVP: готовить текст сообщения, но не отправлять без подтверждения.

### Средние ставки и прогноз цен

Потенциально: получать ориентиры рынка по маршруту и сравнивать их с ответами перевозчиков.

Для MVP: сохранять аналитику в отдельную вкладку Google Sheets.

## 4. Что нельзя считать готовым до проверки

Пока не получены реальные ключи и подтверждение ATI.su, нельзя считать готовыми:

- публикацию грузов;
- отправку сообщений;
- получение контактов;
- сбор ставок из мессенджера;
- автоматический расчёт точных рыночных цен.

## 5. Риски

- API может быть платным.
- Часть методов может быть недоступна текущему аккаунту.
- У мессенджера могут быть ограничения.
- У средних ставок и прогноза цен могут быть отдельные тарифы.
- Массовая отправка сообщений может нарушать правила площадки.
- Browser automation через Playwright рискованнее API и должен использоваться только как запасной вариант с ручным подтверждением.

## 6. Безопасная позиция для MVP

До получения API-доступов агент работает только в dry-run режиме:

- парсит заявку;
- формирует черновик;
- пишет в Google Sheets;
- готовит сообщение перевозчику;
- показывает человеку;
- ничего не публикует;
- ничего не отправляет.
