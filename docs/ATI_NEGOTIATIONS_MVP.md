# MVP переговоров и поиска перевозчиков в ATI.su

## Назначение

Модуль выполняет безопасный цикл поиска перевозчиков, подготовки переписки, получения ставок и согласования перевозок. Поиск и чтение выполняются автоматически, а каждое исходящее сообщение требует подтверждения владельца в MAX.

## Официальные методы ATI, используемые в MVP

### География

- `POST /gw/gis-dict/v1/autocomplete/suggestions` — преобразование названия города в ATI ID.

### Поиск перевозчиков

- `POST /v1.0/trucks/search/by-filter` — поиск свободного транспорта и контактов перевозчиков;
- `POST /v2/dstats/active_firms/search` — поиск фирм, активно работающих или ищущих грузы по направлению.

Оба поисковых сервиса поддерживают `demo=true` для проверки контракта. Боевой доступ является платным и подключается в ATI.

### ATI Messenger

- `POST /messenger/1.1/chats/` — создание диалога;
- `GET /messenger/1.2/subscriptions/` — получение списка диалогов;
- `GET /messenger/1.1/chats/{chat_id}/history/` — получение истории;
- `POST /messenger/1.2/chats/{chat_id}/messages` — отправка сообщения;
- `GET /messenger/1.1/inbox/` — количество чатов с непрочитанными сообщениями.

Текст сообщения отправляется как `multipart/form-data`, а не JSON.

## Реализованный цикл

1. Преобразование названий городов в ATI ID.
2. Формирование запроса поиска свободных машин.
3. Формирование запроса поиска активных перевозчиков.
4. Создание переговорной сессии по перевозчику и маршруту.
5. Формирование первого запроса: актуальность машины, ставка, дата загрузки, форма оплаты и срок доставки.
6. Сохранение истории переписки в SQLite.
7. Разбор ответа перевозчика:
   - доступность машины;
   - сумма и валюта ставки;
   - наличная или безналичная оплата;
   - НДС;
   - срок доставки;
   - дата загрузки.
8. Выбор действия:
   - запросить ставку;
   - задать уточняющие вопросы;
   - предложить встречную ставку;
   - подготовить предварительное принятие условий;
   - корректно завершить переписку.
9. Формирование сообщения через Claude при `ANTHROPIC_ENABLED=true`; при недоступности Claude используется безопасный шаблон.
10. Создание одноразового подтверждения владельца.
11. Создание ATI-диалога и отправка сообщения только после подтверждения.
12. Чтение новых сообщений и запись действий в audit log.

## Жёсткие ограничения безопасности

Реальная запись в ATI не выполняется, если выполняется хотя бы одно условие:

- `DRY_RUN=true`;
- `ATI_MODE` отличается от `APPROVAL_REQUIRED`;
- отсутствует одноразовое подтверждение;
- отсутствует `ATI_ACCESS_TOKEN`;
- отсутствует идентификатор адресата или диалога.

Дополнительно:

- подтверждать сообщения в MAX может только пользователь из `MAX_OWNER_USER_ID`;
- одно подтверждение действует для одного конкретного текста и используется один раз;
- изменение текста после подтверждения требует нового подтверждения;
- внутренние ставки, максимальный бюджет и маржа перевозчику не передаются;
- поисковые методы являются read-only и могут работать при `ATI_MODE=READ_ONLY`;
- `DRY_RUN` блокирует только запись в ATI, но не безопасный поиск и чтение.

## Переменные окружения

```env
DRY_RUN=true
ATI_MODE=READ_ONLY
ATI_ACCESS_TOKEN=
ATI_SEARCH_DEMO_MODE=true

ATI_MESSENGER_CREATE_CHAT_PATH=/messenger/1.1/chats/
ATI_MESSENGER_SUBSCRIPTIONS_PATH=/messenger/1.2/subscriptions/
ATI_MESSENGER_SEND_PATH=/messenger/1.2/chats/{chat_id}/messages
ATI_MESSENGER_HISTORY_PATH=/messenger/1.1/chats/{chat_id}/history/
ATI_MESSENGER_INBOX_PATH=/messenger/1.1/inbox/

ATI_GEO_AUTOCOMPLETE_PATH=/gw/gis-dict/v1/autocomplete/suggestions
ATI_TRUCKS_SEARCH_PATH=/v1.0/trucks/search/by-filter
ATI_ACTIVE_CARRIERS_SEARCH_PATH=/v2/dstats/active_firms/search

MAX_ENABLED=true
MAX_OWNER_USER_ID=

GOOGLE_SHEETS_ENABLED=true
GOOGLE_SHEETS_DRY_RUN=false
DATABASE_URL=sqlite:///./ati_agent.db
```

## Авторизация ATI

Для тестирования можно создать временный токен в разделе «Мои токены» ATI. Его срок действия — 7 дней. Для постоянной интеграции требуется `client_id`, после чего создаётся постоянный `access_token` для отдельного контакта ATI.

Токен хранится только в `/opt/ati-agent/.env`. Его запрещено отправлять в чат, добавлять в GitHub или записывать в audit log.

## Ограничения ATI

Общее ограничение — не более 10 запросов в секунду на контакт. При ответе `429 Too Many Requests` клиент выполняет экспоненциальную задержку: 100, 200, 400 мс и далее.

## Проверка

```bash
export PYTHONPATH=.
pytest -q
```

Тесты проверяют:

- распознавание ставок;
- стратегию торга;
- одноразовые подтверждения;
- блокировку записи в `DRY_RUN` и `READ_ONLY`;
- официальный endpoint отправки ATI;
- формат `multipart/form-data`;
- создание диалога;
- чтение истории;
- автокомплит городов;
- контракты обоих поисковых методов.

## Следующее подключение

1. Безопасно объединить ветку с актуальным серверным кодом `/opt/ati-agent`.
2. Внести временный ATI-токен только в `.env`.
3. Узнать и внести `MAX_OWNER_USER_ID` владельца.
4. Запустить demo-поиск по одному маршруту.
5. Подключить уведомление владельцу в личный чат MAX.
6. Добавить кнопки «Отправить», «Изменить», «Отклонить».
7. Провести dry-run диалог с одним перевозчиком.
8. После проверки перевести только один тестовый диалог в `APPROVAL_REQUIRED`.
9. Сохранять подтверждённые ставки в Google Sheets.
