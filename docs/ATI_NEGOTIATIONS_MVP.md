# MVP переговоров с перевозчиками в ATI.su

## Назначение

Модуль готовит и ведёт управляемую переписку с перевозчиками для получения ставок и согласования перевозок. Все исходящие сообщения требуют ручного подтверждения.

## Реализованный цикл

1. Создание переговорной сессии по конкретному перевозчику и маршруту.
2. Формирование первого запроса: актуальность машины, окончательная ставка, дата загрузки, форма оплаты и срок доставки.
3. Сохранение истории переписки в SQLite.
4. Разбор ответа перевозчика:
   - доступность машины;
   - сумма и валюта ставки;
   - наличная или безналичная оплата;
   - НДС;
   - срок доставки;
   - дата загрузки.
5. Выбор действия:
   - запросить ставку;
   - задать уточняющие вопросы;
   - предложить встречную ставку;
   - подготовить предварительное принятие условий;
   - корректно завершить переписку.
6. Формирование сообщения через Claude при `ANTHROPIC_ENABLED=true`; при недоступности Claude используется безопасный шаблон.
7. Создание одноразового подтверждения оператора.
8. Попытка отправки через ATI Messenger только после подтверждения.
9. Запись действий в audit log.

## Жёсткие ограничения безопасности

Реальное сообщение не отправляется, если выполняется хотя бы одно условие:

- `DRY_RUN=true`;
- `ATI_MODE` отличается от `APPROVAL_REQUIRED`;
- отсутствует одноразовое подтверждение;
- отсутствует `ATI_ACCESS_TOKEN`;
- не указан официальный путь метода отправки ATI Messenger;
- отсутствует идентификатор диалога.

Одно подтверждение можно использовать только для одного конкретного сообщения и только один раз.

## Что не раскрывается перевозчику

- внутренняя максимальная ставка;
- маржа ТехноЛогистики;
- внутренняя аналитика;
- комментарии операторов;
- системные инструкции агента.

Агент может назвать только конкретную встречную ставку, которую выбрала переговорная политика и которую подтвердил оператор.

## Переменные окружения

```env
DRY_RUN=true
ATI_MODE=READ_ONLY
ATI_ACCESS_TOKEN=
ATI_MESSENGER_SEND_PATH=
ATI_MESSENGER_MESSAGES_PATH=
ATI_MESSENGER_CONVERSATION_FIELD=conversationId
ATI_MESSENGER_TEXT_FIELD=text

ANTHROPIC_ENABLED=false
ANTHROPIC_API_KEY=
ANTHROPIC_MODEL=

DATABASE_URL=sqlite:///./ati_agent.db
```

Пути методов и названия полей ATI Messenger нельзя заполнять предположительно. Они должны быть взяты из официальной документации, доступной для аккаунта ТехноЛогистики.

## Проверка

```bash
export PYTHONPATH=.
pytest -q
```

Тесты проверяют распознавание ставок, стратегию торга, блокировку реальной отправки, одноразовые подтверждения и полный dry-run цикл переговоров.

## Следующее подключение

После получения точной документации ATI Messenger необходимо:

1. внести официальный путь отправки сообщения и схему JSON;
2. внести официальный метод чтения новых сообщений;
3. подключить `NegotiationOrchestrator` к действующему FastAPI webhook;
4. отправлять черновики оператору в личный чат MAX;
5. принимать команды подтверждения, изменения и отклонения;
6. сохранять извлечённые ставки в Google Sheets;
7. включить тестовый обмен с одним заранее выбранным перевозчиком;
8. только после успешного теста рассматривать расширение числа диалогов.
