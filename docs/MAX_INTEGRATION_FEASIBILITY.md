# Отчёт о реализуемости интеграции с MAX Messenger

## 1. Вывод

MAX Messenger подходит для первого этапа ATI-агента. Бот может быть добавлен в группу, получать сообщения, видеть текст заявки, отвечать в группу и отправлять кнопки подтверждения.

## 2. Как получать сообщения

Для production-режима предпочтителен webhook. Бот должен получать события о новых сообщениях, извлекать текст, автора, дату, ID сообщения и chat_id.

Long polling можно использовать только для локальных тестов.

## 3. Поиск заявок

Агент должен реагировать только на сообщения, где есть ключевое слово:

```text
ЗАЯВКА
```

После этого он парсит маршрут, автомобиль, дату готовности, состояние, оплату и комментарий.

## 4. Ответы и кнопки

Бот может отправлять сообщения в группу и в будущем показывать кнопки:

- Подтвердить черновик;
- Отклонить;
- Нужны уточнения;
- Передать менеджеру.

## 5. Права бота

Для MVP боту нужны права:

- быть добавленным в группу;
- читать сообщения;
- отправлять сообщения.

Права администратора на старте не нужны.

## 6. Ограничения и безопасность

- Использовать HTTPS webhook.
- Не передавать токен через query-параметры.
- Передавать токен через заголовок Authorization.
- Не отправлять сообщения без DRY_RUN-контроля и ручного подтверждения.

## 7. Что можно взять из опыта Аюба

- Подход к MAX webhook.
- Использование Authorization header.
- Логику обработки входящих сообщений.
- Осторожный режим: черновик сначала человеку, действие только после подтверждения.

Код Аюба при этом не трогаем.
